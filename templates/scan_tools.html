⚙ OUTILS CUSTOM — Exécution shell réelle sur votre machine
Le champ Commande est exécuté directement en shell. Utilisez des placeholders pour contrôler quelle partie de la cible est injectée. Si la cible contient une URL (https://…), chaque outil reçoit automatiquement le bon fragment.
PLACEHOLDERS DISPONIBLES — cible : https://192.168.1.1:5000/api
{target} cible telle que saisie → https://192.168.1.1:5000/api {host} hostname / IP seul → 192.168.1.1 {port} port seul → 5000 {base_url} URL sans le path → https://192.168.1.1:5000 {host_port} host:port → 192.168.1.1:5000
EXEMPLES
nmap → passe le host auto si URL, sinon la cible brute nmap -sV {host} → toujours l'IP/hostname (recommandé pour les scanners réseau) nikto -h {base_url} → URL complète avec port pour les scanners web gobuster dir -u {base_url} → URL de base pour la découverte de répertoires sqlmap -u {target} → URL complète avec path pour SQLMap curl -I {base_url} → requête HTTP sur la base URL
⚠ Sans placeholder, si la cible est une URL le hostname est extrait automatiquement pour les outils réseau. Timeout : 5 minutes.
